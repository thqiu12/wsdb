# SchoolCore Document / Ledger Template Catalog v1

This catalog records documents, ledgers, rosters, and result forms that SchoolCore should support through template generation or structured output.

## 1. System Numbering Settings

| Item | Category | Notes |
|---|---|---|
| 帳票番号設定 | System setting | Numbering rules for certificates, receipts, reports, ledgers, and forms |

帳票番号設定 should support:

- Document type
- School/campus
- Issue date
- Admission period
- Academic year
- Sequence number
- Prefix/suffix
- Reset rule by year/month/document type

Example:

```text
CERT-2026-0001
RC-20260420-202610-0001
ATT-2026-04-001
```

## 2. Immigration / Visa-Related Forms

| Item | Category | Notes |
|---|---|---|
| 資格外活動（上陸） | Visa / Immigration | Permission for activity other than permitted under status of residence, at landing |

System requirements:

- Auto-fill student/applicant identity
- Auto-fill passport and planned entry information
- Store generated file
- Track staff issue/generation record
- Link to COE/pre-arrival workflow

## 3. Admission Documents

| Item | Category | Notes |
|---|---|---|
| 入学決定通知書 | Admission | Generated after acceptance/enrollment decision |

Requirements:

- Select template by school/campus
- Auto-fill applicant name, admission term, course, issue date
- Store generated file in applicant archive
- Record generated by/time/template version

## 4. Graduation / Completion Certificates

| Item | Category | Notes |
|---|---|---|
| 卒業証書 | Certificate | Graduation diploma/certificate |
| 修了証書 | Certificate | Completion certificate |
| 証書台帳 | Ledger | Certificate issuance ledger |

Requirements:

- Auto-generate certificate number
- Issue and void/reissue history
- Link to student record
- Store issuer and issue date/time
- Output printable document
- Certificate ledger can be filtered/exported

## 5. Attendance Books and Rosters

| Item | Category | Notes |
|---|---|---|
| 出席簿 | Attendance | Existing attendance book format |
| 出席簿（新） | Attendance | New attendance book format |
| クラス名簿 | Roster | Existing class roster format |
| クラス名簿（新） | Roster | New class roster format |
| 全クラス名簿 | Roster | All-class roster |

Requirements:

- Output by class, month, term, or academic year
- Support old and new formats simultaneously
- Auto-fill class, teacher, student number, name, nationality, status
- Attendance book should pull attendance sessions/records
- Roster should reflect current enrollment/class assignment
- Export to Excel/PDF depending on template

## 8. Database Additions Suggested

### document_numbering_rules

| Column | Notes |
|---|---|
| id | Primary key |
| school_id | School |
| document_type | Certificate, receipt, report, roster, exam notice |
| prefix_pattern | Prefix rule |
| sequence_scope | Year/month/admission period/document type |
| next_sequence | Next number |
| reset_rule | Annual/monthly/never |
| is_active | Active flag |

### exams

| Column | Notes |
|---|---|
| id | Primary key |
| school_id | School |
| exam_type | Regular, EJU, JLPT, other |
| name | Exam name |
| academic_year | Year |
| term | Term |
| exam_date | Date |

### exam_results

| Column | Notes |
|---|---|
| id | Primary key |
| exam_id | Exam |
| student_id | Student |
| subject_name | Subject |
| score | Score |
| grade | Grade |
| result_status | Passed/failed/etc. |

### certificate_ledgers

May be implemented as a view/export from `certificate_issuances`, or as a dedicated table if legal/operational requirements need immutable ledger rows.
